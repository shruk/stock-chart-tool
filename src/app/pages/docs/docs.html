Scheduled Jobs

All jobs run as Azure Functions timer triggers. They can also be triggered manually via HTTP POST from the Jobs page.

Job Schedule (UTC) Cron Route Notes
Daily Price & Analyst Fetch 9:30 PM 0 30 21 * * * POST /api/jobs/run/fetch-stock-data Backfills missing price bars and refreshes analyst data for all symbols. Runs in background.
Market Summary (AI) 10:00 PM 0 0 22 * * * POST /api/jobs/run/market-summary Generates bilingual EN/ZH market overview using Groq AI.
Risk Calculation 11:00 PM 0 0 23 * * * POST /api/jobs/run/calculate-risks Monte Carlo simulation (2W / 1M / 3M / 6M) for all tracked symbols.
AI Insights Refresh 8:00 AM 0 0 8 * * * POST /api/jobs/run/qa-refresh Fetches FMP data and generates AI Q&A answers (Q1–Q5) for all type=stock symbols. Runs in background.

HTTP API Endpoints

Base URL: https://stock-chart-tool-func.azurewebsites.net/api

Method Route Description
GET /symbols Returns all tracked symbols with bar count, date range, and hasAnalyst flag.
GET /chart/{symbol} OHLCV price bars for a symbol (used by stock chart).
GET /analyze/{symbol} Returns cached AI analyst narrative from analyst_cache.
GET /risk/{symbol} Returns Monte Carlo risk data (2W/1M/3M/6M loss probability and VaR 95%).
GET /market-summary Returns latest bilingual EN/ZH market summary.
GET /qa/{symbol} Returns AI Insights Q1–Q5 data: overview, earnings, peers, management, sentiment.
GET /jobs/status Returns running state, last-run timestamps, and schedule info for all jobs.
GET /logs?type=jobs|admin|ui&hours=48 Queries Azure Application Insights via managed identity. Returns up to 300 log entries (timestamp, level, operation, message). hours range: 1–168.
POST /jobs/run/fetch-stock-data Manually trigger the daily price fetch job.
POST /jobs/run/market-summary Manually trigger market summary generation.
POST /jobs/run/calculate-risks Manually trigger Monte Carlo risk calculation.
POST /jobs/run/qa-refresh Manually trigger AI Insights Q1–Q5 refresh. Returns 202 immediately; runs in background.
POST /symbols Add a new symbol to tracking.
DELETE /symbols/{symbol} Remove a symbol from tracking.

Admin App (Angular)

Angular 17+ standalone component app deployed to Azure Static Web Apps. Uses signal-based state management throughout. Supabase Auth for admin-only access.

Dashboard /

Main view showing all tracked symbols as stock chart cards. Displays price chart (OHLCV), analyst rating badge, and quick stats per symbol.

  • Custom watchlist — user can add/remove symbols from their personal list
  • Risk chart per symbol (2W/1M/3M/6M loss probability bars)
  • Bilingual market summary (EN/ZH) shown at top
  • Test Mode — fake data overlay for demo purposes
  • Click any symbol card → Stock Detail page
Stock Detail /stock/:symbol

Full-page view for a single symbol with all available data panels stacked vertically.

  • Large OHLCV candlestick / bar chart
  • Analyst panel — price target, consensus gauge, recommendation breakdown, next earnings date
  • Risk panel — Monte Carlo results for 4 horizons
  • QA panel — AI Insights Q1–Q5 (company overview, earnings chart, peer table, management, sentiment gauge)
Symbol Manager /admin

Admin-only tool to manage the tracked symbol list. Grouped into default indices and user-added stocks.

  • Add new symbols (stock, ETF, index) with type selection
  • View bar count, date range, and analyst data status per symbol
  • Remove symbols
  • Analyst last refresh time displayed per symbol
Jobs /jobs

Admin-only dashboard for monitoring and manually triggering the four backend jobs.

  • Shows schedule time in user's local timezone (converted from UTC)
  • Active/Inactive status badge for each job's Azure timer trigger
  • Last run timestamp (local time)
  • Run Now button — triggers the job via HTTP POST
  • Live elapsed timer for long-running jobs (polling every 5s)
Docs /docs

System documentation for the entire stack, separated by Functions (backend) and UI Apps (frontend).

Logs /logs

Live log viewer pulling from Azure Application Insights. Three tabs cover different scopes of the system.

  • Jobs tab — traces and exceptions from the 4 job function executions (QaRefresh, FetchStockData, MarketSummary, CalculateRisks)
  • Admin tab — all other Function app traces, HTTP requests, and exceptions (API calls from admin and UI apps)
  • UI tab — browser telemetry (requires Application Insights SDK in the frontend)
  • Time range selector: 6h / 24h / 48h / 3d / 7d
  • Severity badges: Verbose / Info / Warning / Error / Critical — error rows highlighted red
  • Backend uses managed identity to query App Insights REST API — no key in frontend

Mobile App (React Native / Expo)

React Native app using Expo Router for file-based navigation. Deployed via Expo. Calls the same Azure Functions API as the admin app.

Watchlist app/index.tsx

Shows the user's personal watchlist of symbols. Each row displays ticker, company name, latest price, and price change.

  • Add/remove symbols from watchlist via symbol search
  • Pull to refresh
  • Tap a symbol → Stock Detail
Stock Detail app/stock/[symbol].tsx

Full detail page for a symbol showing all five AI Insight sections plus price data.

  • Q1: Company overview text card
  • Q2: Earnings — revenue bars, net income bars, EPS beat/miss table, AI commentary
  • Q3: Peer comparison table (horizontal scroll) with AI-selected metrics + verdict
  • Q4: Management & guidance narrative
  • Q5: Sentiment gauge (0–100) + 4 signal cards + AI commentary
